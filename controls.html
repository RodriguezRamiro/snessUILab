Input Flow Architecture

HARDWARE LAYER (Keyboard/Gamepad)
↓
INPUT MANAGER (modules/input.js)
↓
KEY STATE MAP (keys["w"] = true)
↓
GAME LOGIC LOOP (delta calculation)
↓
UI FEEDBACK (data-key attributes)

The system utilizes a persistent input map. Unlike standard browser text input, the engine tracks the state of a key (held vs released), allowing for smooth, simultaneous physics updates.

Gamepad Auto-detection

Up detection of a gamepadconnected event, the system initializes updateGamepad(). The mapping follows the standard SNES layout:

Standard Mapping:
- Face Button A -> K
- Face Button B -> L
- Start Button -> ENTER

Special Commands

Konami Code Detection

The system monitors the sequence buffer for the legacy override command:

↑ ↑ ↓ ↓ ← → ← → B A

Triggering handleKonami() unlocks extended system memory or hidden game states.
